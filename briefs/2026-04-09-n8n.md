## What is it?

n8n (pronounced "nodemation") is an open-source-adjacent, self-hostable workflow automation platform targeted at technical teams. It sits in the same category as Zapier and Make but with a developer-first philosophy: you can drop raw JavaScript or Python into any node, call arbitrary REST APIs, build multi-agent AI systems, and deploy on your own infrastructure — all from the same visual canvas. With 182k+ GitHub stars it is the most starred workflow automation project on GitHub, and it is SOC 2 compliant.

The platform blends a no-code visual editor with full code escape hatches. Workflows are composed of "nodes" — pre-built connectors for 500+ services (PostgreSQL, SharePoint, Teams, Outlook, Azure, HTTP, etc.) plus AI-native nodes for LLMs, vector stores, and agents. A template library of 8,500+ community-shared workflows accelerates time-to-value. Critically for Garrick's context, n8n now has **native MCP support on both sides**: it can act as an MCP server (exposing workflows as tools to Claude Desktop or Claude Code) and can consume external MCP tools inside a workflow via the MCP Server Trigger node.

n8n uses a "faircode" license — the source is fully visible on GitHub and the self-hosted Community Edition is free, but commercial features (SSO, Git version control, environments, RBAC) require a paid Business or Enterprise plan. The hosted cloud offering starts at €20/month.

## Why it matters (or doesn't)

There are three specific intersection points with Garrick's current work:

**1. CostEstDB / PostgreSQL MCP bridge.** n8n ships a community template called "Build your own PostgreSQL MCP server" that wraps a Postgres database as an MCP tool callable from Claude Desktop. This is almost exactly the architecture CostEstDB needs — a natural language interface to pgvector over MCP — and n8n could provide it with minimal custom code. The workflow handles query routing, scoping, and Claude Desktop auth out of the box.

**2. Plan review & crash report automation.** n8n's webhook triggers, file handling, and AI agent nodes could replace or augment the Azure Functions + Power Automate pipelines currently used for document processing. The "human-in-the-loop" approval step feature (pause a workflow until a human confirms an AI action) maps directly to Abonmarche's compliance posture around municipal documents.

**3. Claude / MCP orchestration layer.** The MCP Server Trigger node lets you publish any n8n workflow as an MCP tool, making it callable from Claude Desktop or Claude Code. This means n8n could serve as the **orchestration glue** between Claude and Abonmarche's enterprise backends (SharePoint, ArcGIS REST, PostgreSQL) without bespoke MCP server development for each system.

The primary tension is with the existing Power Automate investment. Power Automate is deeply embedded in M365 and excellent for simple, user-facing automations. n8n is stronger for developer-built, AI-native, multi-step pipelines where Power Automate hits walls (complex branching, code nodes, multi-agent AI, PostgreSQL vector queries). They are more complementary than competitive — Power Automate for citizen-developer workflows, n8n for the Digital Solutions team's engineering-grade automations.

## Technical details

- **Licensing:** Faircode (source-available). Community Edition is free to self-host; Business plan required for SSO, Git version control, isolated environments, RBAC.
- **Pricing (cloud):** Starter €20/mo (2,500 executions), Pro €50/mo (custom executions), Business €667/mo (40K executions, self-hosted). Enterprise: custom. Billed per full workflow execution, not per step — materially cheaper than Zapier/Make for complex workflows.
- **Self-hosting:** Docker-based, runs on any VM. Azure Container Apps or Azure VM deployment is straightforward. Data stays in your infrastructure.
- **MCP support:** Native MCP Server Trigger node (exposes n8n workflows as MCP tools to Claude, Cursor, etc.). Also documented Claude Desktop integration for connecting Claude to an n8n instance. A third-party MCP server (`n8n-mcp` on GitHub, 1,250+ stars) gives Claude deep knowledge of all 1,396 n8n nodes for AI-assisted workflow building.
- **AI nodes:** LangChain-based AI agent node, vector store connectors, RAG pipeline templates, multi-agent support, output parsers, human-in-the-loop approval steps.
- **Integrations relevant to Abonmarche:** PostgreSQL, Microsoft Teams, SharePoint, Outlook, Azure, HTTP Request (any REST API including ArcGIS REST), OneDrive, Power BI.
- **Maturity:** Production-grade. Used by Vodafone (saved £2.2M in SecOps), Meta, Wayfair, Zendesk. 200k+ community members, active release cadence.
- **Limitation:** Business/Enterprise pricing is steep for an org already paying for Power Automate. Community Edition lacks SSO and RBAC — fine for the Digital Solutions team, problematic for org-wide rollout.

## Recommendation

**Watch for now, with one concrete experiment to validate.** n8n is not an urgent replacement for anything in the stack, but the PostgreSQL MCP server template is worth a direct test against CostEstDB within the next sprint cycle.

Specific next steps:
1. **Proof-of-concept:** Spin up n8n Community Edition in Docker on a local machine or Azure VM (free, ~15 minutes). Load the "Build your own PostgreSQL MCP server" template, point it at the CostEstDB dev instance, and connect Claude Desktop. Test natural language queries against the cost estimating data — this validates whether n8n can replace bespoke MCP server code for database integrations.
2. **Compare to Power Automate** on one real plan-review workflow. Build the same pipeline in both tools and compare: development time, maintainability, AI capability, and execution cost. This creates concrete data for any future platform consolidation conversation.
3. **Track the MCP Server Trigger maturity.** Multi-webhook-replica support for MCP is still rough (single-instance requirement is a scaling constraint). Watch n8n's release notes for improvements here before committing it to a production MCP architecture.
4. **Do not purchase.** Community Edition is sufficient for evaluation and small-team engineering use. Revisit Business plan only if the POC demonstrates clear workflow volume that justifies the cost alongside the existing Power Automate licensing.
