# Make.com

**Category:** automation
**Relevance Score:** 5

## What is it?

Make (formerly Integromat) is a cloud-based, visual workflow automation platform that lets users build multi-step automated workflows — called "scenarios" — by connecting apps through a drag-and-drop canvas. It competes directly with Zapier and Power Automate, positioning itself as the more powerful middle ground: more visual and accessible than code-heavy tools, but far more flexible than simple linear automation platforms. It now counts 3,000+ pre-built app integrations and has been acquired by Celonis.

In 2025, Make significantly leveled up its AI story: it launched Make AI Agents (goal-driven LLM automation modules in beta), a native MCP Server (expose Make scenarios as callable tools to Claude, ChatGPT, Cursor, etc.), an MCP Client (call external MCP tools from within scenarios), and an AI Toolkit. It also shifted from an operation-based billing model to a credit-based model in August 2025. Pricing ranges from free (1,000 credits/mo) to Core at $9/month (10k credits), Pro at $16/month, Teams at $29/month, and Enterprise at custom pricing. MCP Server access is included on all plans, including free.

Make's visual canvas — showing branching logic, routers, parallel paths, and data flow arrows all at once — is widely regarded as the best in class among no-code automation tools. It supports JavaScript and Python code execution, custom webhooks, and REST API modules, making it genuinely usable for technical builders without requiring full code.

## Why it matters (or doesn't)

The single most relevant feature for Garrick's stack is the **Make MCP Server**. It allows any MCP-compatible AI (Claude, ChatGPT, Cursor) to call Make scenarios as structured tools — meaning you could build a Make scenario that, say, pulls from a PostgreSQL database or posts to a SharePoint list, and expose it as a Claude tool without writing a custom MCP server from scratch. This is a meaningful capability for CoP members who aren't comfortable building custom MCP servers in Python.

However, there's a critical context problem: **Abonmarche is a Microsoft shop**. Power Automate is already licensed through M365 and is deeply native to SharePoint, Teams, Outlook, and Dynamics workflows. For any automation that stays inside the Microsoft ecosystem — document approvals, Teams notifications, SharePoint triggers, Outlook routing — Power Automate wins without a fight. Make adds the most value when workflows cross out of M365 into non-Microsoft territory (e.g., bridging ArcGIS REST APIs, VoyageAI, or external PostgreSQL triggers into a Microsoft-connected pipeline).

Compared to **n8n** (already researched, verdict: watch, score: 7), Make is less powerful for engineering-grade use cases: no self-hosting option, no open-source codebase, weaker API-first development patterns. But it is significantly more accessible for non-technical team members on the CoP. If Garrick is building an AI CoP where non-developers need to spin up their own integrations, Make's visual builder is more approachable than n8n's.

For Garrick personally, the MCP Server capability is the most compelling hook — but since he already builds custom MCP servers (CostEstDB), the incremental value of Make's MCP scaffolding is modest.

## Technical details

- **Billing:** Credit-based since August 2025. 1 credit = 1 module action. AI token usage billed separately on built-in AI provider; custom API keys (Anthropic, OpenAI) on paid plans charge credits per operation only.
- **Plans:** Free (1k credits, 2 active scenarios, 15-min min interval) → Core $9/mo (10k credits, unlimited scenarios, 1-min interval, Make API access) → Pro $16/mo → Teams $29/mo → Enterprise (custom, SSO, 24/7 support, audit logs, overage protection)
- **MCP Server:** Included on all plans. Connects Claude, ChatGPT, Cursor and other MCP-compatible agents to Make scenarios. No local server setup required. Scenarios become structured tools with typed inputs/outputs.
- **Code execution:** JavaScript and Python supported via Make Code App on paid plans (2 credits/sec of execution time)
- **Security:** GDPR, SOC 2 Type II, SOC 3 compliant. SSO (OAuth2/SAML2) on Enterprise. Hosted on AWS (EU/North America).
- **Limitations:** No self-hosting. No RPA/desktop automation (unlike Power Automate Desktop). Steep learning curve — Make's own support team reportedly recommends ~19 hours of Make Academy training before building production workflows. Credit model creates potential for surprise overage costs at scale.
- **Owned by Celonis** (process mining company), which adds enterprise credibility but introduces some strategic uncertainty about product direction.

## Recommendation

**Watch — don't pursue actively right now, but revisit if two things happen:**

1. **Power Automate hits a wall on a specific workflow** — if a CostEstDB, crash report, or plan review pipeline needs to connect to a non-Microsoft service in a way that's painful to build in Power Automate, Make is the fastest no-code bridge.
2. **CoP members want to build their own Claude integrations** — Make's MCP Server is a legitimate low-barrier path to letting non-developers create Claude-connected workflows without writing custom MCP server code. If that use case emerges in the CoP, a Make Core account at $9/month is a low-risk experiment.

Do not prioritize Make over n8n for engineering-grade pipelines. n8n is more powerful, self-hostable, and better suited to Garrick's developer profile. For now, Power Automate + custom Claude MCP servers covers the core territory adequately.
