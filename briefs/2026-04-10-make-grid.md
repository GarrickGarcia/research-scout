## What is it?

Make Grid is a feature of Make.com (formerly Integromat) that provides an automatically generated, real-time visual map of an organization's entire automation and AI landscape. First unveiled at Make's annual Waves event in October 2024 and launched into Open Beta on June 24, 2025, Grid was initially conceived as a read-only dependency map — a bird's-eye view of how scenarios, apps, data stores, and AI agents interconnect across an organization's Make environment.

The "map to workspace" evolution referenced in the prompt describes Make Grid's planned progression beyond passive visualization into active territory: annotating the map, tweaking elements directly from Grid, organizing automation folders, and collaborating with teammates — all without leaving the landscape view. Make's product team has stated their long-term vision is for Grid to become "a hub for building and adjusting your entire automation landscape," eventually powered by natural language AI commands.

At its core today, Grid updates automatically (approximately every two minutes), shows error indicators for failing scenarios, surfaces dependency chains (which scenarios rely on a shared webhook, database, or API), and supports filter/search to isolate specific apps or flows. It is included in all paid Make plans starting at $9/month (Core tier).

## Why it matters (or doesn't)

Abonmarche already has Make.com on its radar (see: `2026-04-09-make-com.md`, watch, score 5), but Make Grid warrants its own note because it addresses a very specific pain point Garrick will face as Abonmarche's automation footprint grows: **governance and visibility across a sprawling set of workflows**.

Whether the automation stack ends up being Power Automate, Make, n8n, or a hybrid, the underlying problem Make Grid is solving — "what breaks if I change this?" and "where does data actually flow?" — is universal. Grid's dependency mapping and real-time error surfacing are directly analogous to what a good infrastructure diagram does for GIS or Azure architecture. As Garrick scales the AI CoP and more staff build their own automations, a tool like this becomes operationally critical.

The **Make MCP Server** (a separate but co-released capability) is the more immediately actionable angle: it exposes Make scenarios as callable tools for Claude, ChatGPT, or any MCP-compatible AI — included in all plans, no extra cost. If any CoP member is already running Make scenarios, Claude Code can call them directly via MCP today. Grid helps manage the growing library of those scenarios.

That said, if Abonmarche is not a Make.com shop, Make Grid offers little standalone value — it only maps Make-native scenarios, not Power Automate flows, Azure Functions, or ArcGIS webhooks.

## Technical details

- **Status:** Open Beta (free to access for all Make users as of June 24, 2025)
- **Update cadence:** Near real-time (auto-refreshes ~every 2 minutes)
- **Pricing:** Included in all paid plans; Core plan starts at $9/month (10,000 credits/month); Free plan available (1,000 credits/month, limited scenarios)
- **Make transitioned from operations to credits in August 2025**; AI provider connections (including Anthropic/Claude API keys) are now available on all paid tiers as of November 2025
- **Make MCP Server:** Also included in all plans — turns Make scenarios into callable tools for Claude and other MCP-compatible AI agents; no local server required
- **Make AI Agents:** Also in beta alongside Grid; Make scenarios can be exposed as agentic tools
- **Limitations:** Grid only visualizes Make-native objects (scenarios, data stores, webhooks, AI agents). Does not integrate with Power Automate, n8n, Azure, or ArcGIS workflows. "Workspace" editing features (annotation, in-place editing, AI-generated descriptions) are still on the roadmap — not fully shipped yet.
- **Compliance:** SOC 2 Type II, GDPR compliant; SSO on Enterprise tier
- **Maturity:** Early-stage feature with real customer validation (ChargeGuru reported 2-hour/day debugging savings; CMCC Foundation cut workflow mapping time by 50%)
